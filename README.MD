## План рефакторинга на Clean Architecture

### Этапы рефакторинга

1. **Создать Domain слой** — выделить доменные сущности (entities) без зависимостей от фреймворков, определить интерфейсы репозиториев

2. **Создать Use Cases** — инкапсулировать бизнес-операции в отдельные классы (один use case = одна операция)

3. **Разделить Data слой** — создать отдельные data source интерфейсы для удалённых и локальных источников данных

4. **Создать маппинг моделей** — реализовать мапперы для преобразования между API/DB моделями и доменными сущностями

5. **Реализовать репозитории** — имплементировать интерфейсы репозиториев из domain слоя, координируя работу data sources

6. **Выделить Presentation слой** — отделить UI логику от бизнес-логики, использовать use cases вместо прямого обращения к репозиториям

7. **Реорганизовать пакеты** — структурировать код по слоям: domain/, data/, presentation/

8. **Покрыть тестами** — написать unit-тесты для domain и use cases, интеграционные тесты для data слоя

9. **Провести код-ревью** — проверить соблюдение принципов Clean Architecture и устранить нарушения
